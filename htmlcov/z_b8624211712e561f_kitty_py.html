39class KittyImage(GraphicsImage): 

40 """A render style using the Kitty terminal graphics protocol. 

41 

42 See :py:class:`GraphicsImage` for the complete description of the constructor. 

43 

44 | 

45 

46 **Render Methods** 

47 

48 :py:class:`KittyImage` provides two methods of :term:`rendering` images, namely: 

49 

50 LINES (default) 

51 Renders an image line-by-line i.e the image is evenly split across the number 

52 of lines it should occupy. 

53 

54 Pros: 

55 

56 * Good for use cases where it might be required to trim some lines of the 

57 image. 

58 

59 WHOLE 

60 Renders an image all at once i.e the entire image data is encoded into one 

61 line of the :term:`rendered` output, such that the entire image is drawn once 

62 by the terminal and still occupies the correct amount of lines and columns. 

63 

64 Pros: 

65 

66 * Render results are more compact (i.e less in character count) than with 

67 the **LINES** method since the entire image is encoded at once. 

68 

69 The render method can be set with 

70 :py:meth:`set_render_method() <BaseImage.set_render_method>` using the names 

71 specified above. 

72 

73 | 

74 

75 **Style-Specific Render Parameters** 

76 

77 See :py:meth:`BaseImage.draw` (particularly the *style* parameter). 

78 

79 * **method** (*None | str*) → Render method override. 

80 

81 * ``None`` → the current effective render method of the instance is used. 

82 * *default* → ``None`` 

83 

84 * **z_index** (*int*) → The stacking order of graphics and text for 

85 **non-animations**. 

86 

87 * An integer in the **signed 32-bit** range (excluding ``-(2**31)``) 

88 * ``>= 0`` → the image will be drawn above text 

89 * ``< 0`` → the image will be drawn below text 

90 * ``< -(2**31)/2`` → the image will be drawn below cells with non-default 

91 background color 

92 * *default* → ``0`` 

93 * Overlapping graphics on different z-indexes will be blended (by the terminal 

94 emulator) if they are semi-transparent. 

95 * To inter-mix text with graphics, see the *mix* parameter. 

96 

97 * **mix** (*bool*) → Graphics/Text inter-mix policy. 

98 

99 * ``False`` → text within the region covered by the drawn render output will be 

100 erased, though text can be inter-mixed with graphics after drawing 

101 * ``True`` → text within the region covered by the drawn render output will NOT 

102 be erased 

103 * *default* → ``False`` 

104 

105 * **compress** (*int*) → ZLIB compression level. 

106 

107 * ``0`` <= *compress* <= ``9`` 

108 * ``1`` → best speed, ``9`` → best compression, ``0`` → no compression 

109 * *default* → ``4`` 

110 * Results in a trade-off between render time and data size/draw speed 

111 

112 | 

113 

114 **Format Specification** 

115 

116 See :ref:`format-spec`. 

117 

118 :: 

119 

120 [ <method> ] [ z <z-index> ] [ m <mix> ] [ c <compress> ] 

121 

122 * ``method`` → render method override 

123 

124 * ``L`` → **LINES** render method (current frame only, for animated images) 

125 * ``W`` → **WHOLE** render method (current frame only, for animated images) 

126 * *default* → Current effective render method of the image 

127 

128 * ``z`` → graphics/text stacking order 

129 

130 * ``z-index`` → z-index 

131 

132 * An integer in the **signed 32-bit** range (excluding ``-(2**31)``) 

133 * ``>= 0`` → the render output will be drawn above text 

134 * ``< 0`` → the render output will be drawn below text 

135 * ``< -(2**31)/2`` → the render output will be drawn below cells with 

136 non-default background color 

137 

138 * *default* → ``z0`` (z-index zero) 

139 * e.g ``z0``, ``z1``, ``z-1``, ``z2147483647``, ``z-2147483648`` 

140 * overlapping graphics on different z-indexes will be blended 

141 (by the terminal emulator) if they are semi-transparent 

142 

143 * ``m`` → graphics/text inter-mix policy 

144 

145 * ``mix`` → inter-mix policy 

146 

147 * ``0`` → text within the region covered by the drawn render output will be 

148 erased, though text can be inter-mixed with graphics after drawing 

149 * ``1`` → text within the region covered by the drawn render output will NOT 

150 be erased 

151 

152 * *default* → ``m0`` 

153 * e.g ``m0``, ``m1`` 

154 

155 * ``c`` → ZLIB compression level 

156 

157 * ``compress`` → compression level 

158 

159 * An integer in the range ``0`` <= ``compress`` <= ``9`` 

160 * ``1`` → best speed, ``9`` → best compression, ``0`` → no compression 

161 

162 * *default* → ``c4`` 

163 * e.g ``c0``, ``c9`` 

164 * results in a trade-off between render time and data size/draw speed 

165 

166 |

169 Currently supported terminal emulators are: 

170 

171 * `Kitty <https://sw.kovidgoyal.net/kitty/>`_ >= 0.20.0.
